Introduction

Rusty Trader is a web app that allows players of Rust: Console Edition to browse and create trades for in-game items. Like many of my projects, Rusty Trader was created from a specific problem I had experienced. Trade is an important aspect of the game and the lack of a built-in text chat makes it difficult.

This will be less of an issue once the community is allowed to rent and run game servers. Trade could be facilitated with an external application, most likely a Discord server. This still leaves no option for the game’s official servers run by Double Eleven.

Rusty Trader has an easy-to-use and familiar interface that makes it easy to browse and post trades for in-game items. It is as simple as searching for the server you play on and browsing trades. Creating a new trade brings up an interface that mimics the in-game vending machine GUI so it is familiar and easy to use.

How It Was Built

Frontend

Rusty Trader consists of a front-end web application written in JavaScript and powered by React. It also uses MSAL for React to handle authentication with the backend and the Azure B2C service. I also use React Bootstrap for the main layout of the page and combine this with custom CSS. React Bootstrap let me quickly create a responsive layout and the custom CSS gives the application some character so it doesn’t look like a generic Bootstrap site.

Backend

The backend is a web API written in C# and using the ASP.NET Core framework. I choose these technologies for the backend for several reasons.

The first is simply that I enjoy the C# language. I have used it previously with game development projects and ASP.NET MVC websites. It is strongly typed, feature-rich, and cross-platform. It does everything I need it to do, it is easy to read and understand, and the development environment Microsoft has created is unmatched.

The second reason is that ASP.NET is mature and packed with features. Implementing authentication and authorization was a breeze especially with the Azure B2C service. It was easy to create an API controller that implements all of the CRUD actions for any resource my app uses. Protecting these routes is as easy as adding the [Authorize] attribute above the controller class.

A final reason for using this stack for the backend was to get more familiar with it. C# is growing and is quickly becoming a popular choice with companies when creating an application. I wanted to use the language and the framework for a real project as that is the best way to learn and become more comfortable with it.

Database

The last part of the technology stack I want to discuss is my choice for a database. Instead of using a SQL database, I used MongoDB. There was no architectural reason behind choosing a NoSQL database (although there certainly can be). It was purely to get more experience with NoSQL databases and MongoDB in particular. I will say that using the official MongoDB C# package makes it very easy to work with the database. It was also convenient to quickly prototype without the need to worry about a strict schema or database migrations. MongoDB’s document model maps very nicely to objects (like trades in the case of Rusty Trader). In addition to the ease of use, it was also trivial to set up a free cloud database with MongoDB Atlas.

Wrapping Up + Screenshots

Overall I enjoyed my time spent working on this project. C# is a fantastic language and is already quite popular. I see it growing in popularity even more with the direction Microsoft has been going. I will likely continue to add new features and may get to a point where I feel comfortable deploying it for public use. I am working on a notification system where players can be notified if someone is interested in a trade.
