Introduction

What Lies Below was a project I worked on for a game development class. It is written entirely in C# and is created with the help of the Unity3D game engine.

This project strictly followed SOLID design principles to ensure clean code that was easy to maintain and extend as the project grew. This project grew rapidly as I added new features to the game over about two months. In total I created over 50 C# classes to support a wide range of game systems.

Major Features

  • First Person Character Controller
    • Basic Movement + Jump
    • Health
    • Skills
  • Inventory System
    • Item stacks
    • Comsumables
    • Equipable items
  • Random Loot Spawns
    • Support for different loot tiers
  • Settlement System
    • Claim settlement as home base
    • Collect resources to build upgrades
    • Higher tier settlements support more characters
  • Enemy AI
    • Health
    • Basic Animations
    • Damage the player
    • Supported with a state machine
  • Recruitment System
    • Recruit NPC’s to the player settlement
    • Allowed player to switch to a new character on command

What I Learned

Over the course of the Game Development class I learned a ton of very important lessons.

This was without a doubt the largest project I had created and worked on completely alone. As I kept building out new game features and the project kept expanding I saw first hand the importance of SOLID principles.

Adhering to these 5 principles significantly reduced the amount of code refactoring I needed to do. It also kept the project manageable and prevented me from ever feeling overwhelmed with the size and complexity of the project.

I am also very happy with my decison to do some pre-planning before I started writing even the first line of code. I used Notion which is an amazing tool for project planning, organizing notes, and typical day-to-day planning. Laying out what features I wanted to include allowed me to design these systems before I started writing code.

In addition to these lessons I also found myself much more comfortable using C# and the Unity engine. I prevously used both in a small game I published to the Xbox Store called Dorado. What Lies Below is magnitudes more complex and thus allowed me to get more comfortable with the many features of the language.

Additional Notes

This project was extremely enjoyable and fulfilling. It definitely sparked my interest in game development not only as a hobby but even as a potential career path. After the class was over I was inspired to setup a CD/CI system with Github Actions. I will include more information about that project in another post.
