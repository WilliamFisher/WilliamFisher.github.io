Introduction

Originally I had a very simple setup for my personal website. It is a very basic Bootstrap jumbotron with a link to my Github page. This was only ever meant to be a palceholder until I had the time to create a real website.

My website has looked like the image above since May of 2019. Nearly two years later I have realized it is time for a serious upgrade.

What I Wanted

I knew I wanted a more fully featured website where I could display some basic info about me and showcase some of my personal projects. As I am going into my senior year, I wanted my own website to be a better refelection of my skills.

A boring Bootstrap Jumbotron linking back to my GitHub is not a good display of my degree and experience.

Static Site Generators

My old boostrap website was hosted using Github Pages. I really like the simplicity of hosting a personal website this way.

I could go the route I started with and spend a ton of time building out a static site from scratch. However, I have very little time to dedicate to this and although I have a good feel for what looks good I am not a designer.

For these reasons I decided I wanted to build the site with a static site generator. There are tons of these available. Some of the most popular are Next.JS, Hugo, Gatsby, Jekyll, and Nuxt. I decided to use Hugo to create my site thanks to it’s incredibly fast build times, simplicity, and amazing template system.

Getting Started with Hugo

Hugo is incredibly easy to get started with. If you are using a Mac with Homebrew, installing Hugo is an simple as running brew install hugo in a terminal window.

Creating a new site

After Hugo has been installed, to create a new site it is as easy as running the following command:

hugo new site my-site

This will create a folder called my-site and generate a new Hugo site in the directory. If you do not have time to create a custom theme from scratch, you can pull in one of the hundreds of beautiful themes available for Hugo. I suggest themes.gohugo.io for finding a good high-quality theme.

Creating new posts

After you configure your theme, new posts can be created with hugo new posts/my-first-post.md. This will create a new mockdown file in the post directory and mark it as a draft.

Starting a local dev server

While developing the site you will want to start the built-in webserver. This web server will allow you to preview your site and it has the added benefit of supporting hot-reloading. Run the following command to start up the built in web server:

hugo server -D

The -D flag will tell Hugo to show posts that have been marked as a draft.

Building your site

When you are ready to deploy your new website you must run the build command. Make sure you are in the my-site directory and run the following command:

hugo

This will build the site and place all of the static files in the public/ directory. For my use case I would then copy these files over to Github Pages where they will be served up like any static website.
